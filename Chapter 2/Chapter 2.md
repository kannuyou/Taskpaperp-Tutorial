# 第二章：简单使用

TaskPaper 本质上是一个对列表做了优化的文本编辑器，绝大多数操作用快捷键和打字进行更为高效便捷。从本章的基础操作开始，我主要基于这两点展开讲解。

## 界面概要

【界面】

TaskPaper 的界面简洁，界面大致分为菜单栏（menubar）、侧边栏（sidebar）、搜索栏（searchbar）和编辑界面。

Menubar 内的选项非常丰富，不过出于效率考虑，我建议多用快捷键（大多数操作是有快捷键的）。只要是能用键盘完成的操作，我都会在本教程里注明，并于每篇文末附上表格。

左侧为 sidebar，所有的项目、标签和已保存的搜索项都在罗列于此，一览无余。

编辑界面上方是 searchbar，用于过滤。如果你没看见它，可以用「cmd+shift+f」召唤出来。**在新系统中，召唤 searchbar 后光标不会自动定位，需要连按两次组合键，或者手动点选。**

## 基本元素

TaskPaper 的基本元素只有四个：

```
- 任务

项目:

@标签

备注
```

上手的时候，只需要使用它们就足够了，但如果你太轻视它们，也会遇到麻烦。

新手容易犯的一个错误就是滥用符号，与 TaskPaper 基本元素简洁易用的意旨相悖。

我曾给某知名写作平台投稿，某次自觉满意的作品，突遭拒稿；原来拒稿官换了初学 Markdown 的新人，要求我「多多使用标记符号」。遂拜读此君作品，通篇全是大小标题、引用、无序列表、代码框（里面是打油诗）。这已经成了笑柄，但轮到我们使用 TaskPaper 的时候，起初带着三分钟热度，也容易犯类似错误。

所以，我们有必要知道各个元素的标准用法。

### 任务

在一行文字前标出一个`短横＋空格`即设此行为任务。快捷键为「cmd+return」，如果光标在任务行的末尾，直接回车也可以新建任务。

任务是 TaskPaper 里最基本的单位。

任务之下可设子任务，形成任务组。并列的两个任务，使用快捷键「ctrl+cmd+→」将下方的任务右移一级，便把它设为了上方任务的子任务。移动光标到主任务左侧，会出现空心小红点，点击可折叠/展开子任务列表（快捷键为「cmd+9/0」）。已折叠的任务左侧红点会变成实心，以示其下包含有的已折叠项。

【折叠】

### 项目

在一行文字后标出一个`冒号`即设此行为项目，或者使用快捷键「opt+cmd+return」。项目下可以添加任务，操作与设置子任务类似。所有的项目都会显示在 sidebar 的「Project」栏内，包括子项目（它们可以折叠起来）。

TaskPaper 里有一个特殊项目：Archive。使用快捷键「cmd+shift+A」可以将带有`@done`标签（关于标签详见下一节）的任务归档到 Archive 项目，从此这些任务不再显示于任务列表，只能在 Archive 里看到它们。如果你不想看到那些已完成的任务，可以用这个方式快速归档它们。

【Achieve】

在几个项目之间移动项目（或任务），只需要光标拖拽：按住它们前面的红点，把它拖到你想要的地方——编辑界面、sidebar（侧边栏），或者另一个 TaskPaper 文档。

【拖拽项目】

### 标签

`@` 符号后面的文字即为标签内容，一个普通标签里不能出现空格和 emoji 表情。所有标签都会显示在 sidebar 的「Tags」内，点击可快速查看所有带该标签的任务/项目。

TaskPaper 的文字标签支持灵活的赋值、和运算号组合，进行精确的过滤。过滤对于任务管理是颇具意义的，你可以无视层级，用一个或一组搜索条件从纷杂的列表中快速揪出需要的任务，聚焦于这些事情，不受其他任务打搅。如果任务管理工具只是摆路边摊一样，把任务、项目和标签一股脑堆在我眼前，我反而不知道自己该干什么。

TaskPaper 的标签是纯文字的，也许你会暗自抱怨：居然还要打字，实在是太不方便了。实际上，TaskPaper 的标签支持自动补全，不用次次完整打出。你输入过一次标签之后它会被记忆住，下次打出几个首字母就能显示出来。如果你连「@」也不想打，可以使用「cmd+t」呼出标签栏，同样享受自动补全。

【自动补全】

一般来说，你可以往标签里塞任何文本内容，不会引起程序曲解；但是 TaskPaper 预设了一些标签，它们具有特殊的意义，需要单独拎出来介绍。

#### `@done`

快捷键为「cmd+d」，可将任务画上删除线、标为已完成状态。左侧 serachbar 内置了一个搜索项`not @done`，可以快速过滤出没有完成的任务。如果误按，再按一次快捷键可以将任务重置为未完成的状态。

在 TaskPaper 的 Task 设置里，可以勾选「include date when tagging task with @done」，为已完成的任务附上日期（如果你需要做一个详细的回顾的话）。

#### `@due`和`@start`

它们分别标注任务的截止日期和开始日期。单独的@due或者@start没有什么意义，它们需要配合一个时间点才能起到指示任务具体执行时间的目的。**TaskPaper 用 ISO 8601 格式的文本存储时间 ，这是一个超棒的时间标准，在最后几章的实例中你会看见 TaskPaper 如何导出任务到别的应用里——直接就设置好了起止时间。**但是用该格式输入完整的文本并不方便；为此，你可以使用 TaskPaper 预设的日期&时间语法。日期&时间和`@due`或`@start`标签组合起来就是这样：

```
- 订阅 Ulysess @due（2017-10-22）
- 跑步五公里 @start(2017-10-02)
- 阅读《睡鼠说》@due(today) @start(next Sunday)
```

这两个标签也在 sidebar 上，不过作用有限，只能过滤出「有截止日期的任务」或「有开始日期的任务」。需要精确地过滤一组符合要求的任务出来，还需使用下一章详细介绍的搜索语法。

别紧张，会有模板给你。

#### `@today`

严格来说，`@today`标签本身不具备时间上的意义，它并不表示具体的时间，只是告诉你：这个任务今天要做哦。如果你今天没完成，明天它还是显示一个「today」。而且和 TaskPaper 日期时间语法里的「today」不同，`@today`不会以标准时间格式存储，你把一个标了`@today`的任务导进提醒事项，看见的还是「@today」一串文字，没有变成提醒时间。

【@today】

但`@today`也有用处，比如你有好几个项目，每天早上遍历一次（一一查看），觉得今天可以去做的就打上`@today`（可以用快捷键「cmd+y」），当天只需要依据 sidebar 里的`@today`行事就可以了。Sidebar 里还有一个 Today 的 searches，是默认的搜索项，它汇集了绝对时间意义上的今日任务和你临时打上`@today`标签的任务，完整地把当日任务集中呈现。

这些预设于 sidebar 的标签是可以自己修改的，如果你实在不喜欢 TaskPaper 默认的标签显示方式，或者像我一样觉得`@today`用着不顺手，可以在对应的配置文件里自定义需要现实的标签。在`/Users/你的用户名/Library/Application\ Support/TaskPaper/Configurations/tags.taskpaper`这个路径里就是配置文件，我就把`@today`干掉了。

【tags 配置文件】

#### 补充包：日期语法

鉴于接下来许多标签都用到了 TaskPaper 格式的日期与时间，有必要讲一下它们的语法。TaskPaper 里的日期和时间有两种类型：绝对日期和时间，形如「2017-9-15」「9:00 am」；相对日期和时间，比如「next Wednesday」「+2h」。你也可以组合使用它们，像是「today+24h」。

【日期时间语法】

TaskPaper 的日期和月份可以使用缩写，例如 Saturday 缩写为 Sat。一天的开始是午夜，一周的起始是周末（依据 ISO 8601 标准）。

需要注意的是，相对时间的「坐标点」是当下或当天，而非录入任务时，也就是说，你五个小时前标上的「+5h」，现在它指向的绝对时间还是五小时后，不会自己改变。我建议打标签的时候少单独使用相对时间，多使用绝对时间或把它们组合起来使用，避免歧义；至于搜索，则根据需要灵活选择，择精确而输入量少者用之。

### 备注和附件

如果输入一行文字，不做任何处理，即设此行为备注。对于任务、项目的大段描述，可以用备注的形式添加进 TaskPaper。

至于附件，它本不属于 TaskPaper 的基本元素，但是也起到了附加信息的作用，我把它和备注一起介绍。TaskPaper 里的附件不是真把原文件塞进来了，它只是显示一个可点击跳转的完整路径。虽然不能直接预览，但是具有轻量、直观显示文件真实位置的优点。

添加附件很方便，直接把目标文件拖拽到 TaskPaper 页面即可。

【拖动添加文件】

如果你的备注过长，或者附带了一串附件路径，建议把它移到对应任务的下一层级（快捷键「ctrl+cmd+→」），如此一来，平时可以把它折叠起来。涉及到层级的具体内容会在下一章讲解。

【折叠备注】

## 窗口管理

TaskPaper 可以打开多个窗口或者标签页。

对于同时管理多个复杂项目的使用者，我建议他们把几个项目存储在不同的 TaskPaper 文档里，所需搜索和定位时候的范围。

## 实例：Daily Plan，拒绝拖延

> If a task isn’t written down in a list or set to alert me in iCal, it’s gone. Forgotten. Doesn’t get done.（你不把任务记下来或设个闹钟，它就溜走了、被遗忘了，不会被完成）——Marco Arment（Instapaper、Tumblr 开发者）

任务管理最基础的就是「记录」，无论是用原始的纸质便笺，还是灵活的人物管理应用。了解完 TaskPaper 的基本元素，我通过一个例子来展示它们的简单运用，也阐述我做任务管理的基本思路。

一开始的时候，我对「管理」也并不精通，于是就从「记录」入手；恰好这在许多认为管理体系里都是一个容易做而且用处大的步骤，GTD 中就把「收集」作为第一步，让践行者清空大脑、把任务一个不漏地记在纸上（当然现在我们是记在 TaskPaper 里）。把任务从大脑里腾到纸或其他载体上，有两个优点：

* 不容易遗忘
* 集中注意力

而这一切的意义在早晨更为重大。许多人认为「一日之计始于晨」，对此我有着独特的理解：**在任务管理中，早晨的地位不是开始，而是过渡。**任务管理的周期并不和太阳的起落完全吻合，日出而作、日落而息在今天看来有些落伍（对于养生达人来说也许不是这样），**在我看来早晨的作用是回顾昨日任务、规划当日任务，做一个 Daily Plan，而 TaskPaper 就是快速、有序、结构化完成这一切的利器。**

记录的时候，需要一边打字一边筛选、评估，让最后出来的 Daily Plan 具有较高的可行性。

Daily Plan 里的任务，不是起床后从天而降、凭空捏造的，它们有三个来源：

* 昨日余留任务；
* 躺在床上冒出的灵感；
* 循环任务（对于我是课程）；

所以在这个来自现实的例子里，我一开始就会给你看一堆任务，而不是假惺惺手把手地教你新建。

### 处理遗留

【历史遗留】

首先是来自昨日的任务。**早上再来处理历史遗留任务的根本原因在于避免拖延。**为什么当天不处理，还要拖到翌日清晨？其实晚上做计划是非常低效乃至虚伪的，尤其临睡前，你的意志力往往很弱（精力旺盛不等于意志力强，更不等于效率高，可能你只是白天太压抑了睡前放飞自我）：

>  毁灭人类的方法非常简单，那就是告诉他们还有明天。因为告诉他们还有明天，他们就不会在今天努力了——辰格《戒了吧！拖延症

早起脑子清醒（如果你是夜猫子，请看下一条理由），而且可以把遗留任务和其他来源的任务统筹考虑，不像临睡前的保证那样，毫无用处地许诺「我明天一定会做的」。当天的事，早起规划、（尽全力）当天做完。

虽然不是光荣的事情，但我得承认自己有完不成当日任务的情况。**对于这些任务，我会再做一个必要性和可行性的评估：需不需要做这件事？能不能够做这件事？**取表中二例：昨日中秋，赶路回家，我欠下了两百多个单词没背，这件任务理所当然顺延到今天；而取景的任务，已经足足拖延了半个月，这期间当地不是暴雨就是雾霾（接下来还是阴雨l连连），完全没法拍摄，只好放弃。**TaskPaper 的`@due`标签帮助我找出那些早该完成的任务，它们是需要优先处理的。**

经历了两年的每日清理，我发现一个残酷的事实：如果某个任务连续拖了一周，不是这件事毫无处理的必要，就是我根本没有能力去完成它（前提是执行者不是一个懒汉），无论何者，我都应该无视这个没有意义的任务，把精力放在更可行的事情上。

### 收集灵感和细碎片段

比起「吾日三省吾身」，爬到宿舍上铺之后（或者把自己丢到青旅床上），我更喜欢看点书。反省（回顾）固然重要，但是于我而言，会增加负罪感，影响睡眠质量；清晨时分再和昨日余留「算账」，而一旦躺在床上，我只会想着多充充电，灵感（任务）也随之而来了。我最近在读《论摄影》，书中常常提及一些上世纪的电影名作（大多是我没看过的），例如六十年代的《偷窥狂》，就上了我的观影单。为了避免对睡眠的干扰，我不愿点亮手机屏幕记录任务，而是在床头放了铅笔和便笺，草草记下几个关键词；醒来后，这些堪比魏晋书法作品的纸片，就成了我的一大任务来源。接下来不是机械地把任务输入电脑，而是开始解构任务。

**我的任务基本是「动词+名词」的格式，像`- Search：《偷窥狂》`，这样的显示非常简明，并且记录也方便。**动词很神奇，它本身是带有**先后逻辑**的：我记录在纸上的关键词往往只有名词，而且是专有名词，并不记录动词；因为我到底是搜索、了解、下载还是观看这部电影，写便签时尚不确定，需要录入 TaskPaper 的时候统筹当天任务、再做判断。我在长期的阅读中总结了一个很黑暗的规律：一本书里所提到的另一部作品（只要它不在「中学生必读书目」「人生必看 100 部电影」里），后者的名字在绝大多数情况下是译者兴之所至随意取的，我几乎没有直接成功搜到过。也由此，找资源将是一个不确定耗时、不一定能完成的任务，我添加任务的时候只敢用「Search」，而不是「Download」之类。至于修饰的词语我偶尔会用，比如说《论摄影》这本书，有桑塔格和亚当斯两位大师的同名作，我只在这种特殊情况下使用`- Search 《论摄影》苏珊`的格式。

但不是所有的任务都需要如此循规蹈矩。回忆一下我在第一章里的例子，用 TaskPaper 记录当日所需商谈的客户名单。由于我是个不关心公司其他事务的创始人，所有涉及公司的事情，基本就是被动地谈客户，那么「谈」这个动作就可以缺省了；原先属于备注的产品名，因为简短而具体，此时就代替客户名成为任务中的谓语对象。

**总之，我记录任务，一般只需要最核心的动作和对象。**如此，在 check 任务列表的时候，我的注意力可以集中在最重要的元素上，「Just do it」。用 TaskPaper 记录这样「格式化」的任务很棒——还是回到纯文本，它显示的方式直观而无多余元素，看着清清爽爽的列表我就感到动力十足。

> Keep a to-do list……A real one. One that you actually use and update throughout the day. It doesn’t need to be fancy, like the getting-things-done task managers — I use TaskPaper, which is essentially a text editor with optimized syntax highlighting for to-do lists, against a text file on Dropbox.（做个你真正在用的、每天更新的清单，它不用像 GTD 管理工具那么酷炫——我用的就是对于清单显示方式有优化的 TaskPaper，并把它存在 Dropbox 里）——Marco Arment

### 拆开「日复一日」的任务

我把大量时间花在处理由灵感带来的任务上，但我相信这不是长久之计。我算半个搞艺术的（设计师总误认自己是艺术家，而后者很少以设计师自居），现在年轻，尚可一年到头智力火花四溅，可惜我不是天才，必须持续地学习和练习。

我在学校蹭了好几门需要的课程，上课就是每日「循规蹈矩」的任务。它们可以通过时间块的方式展现，比如写进日历、设置循环——不过我更加喜欢用 TaskPaper「分拆」它们。

例如每隔两天就有刑法课（来自我的主修专业），但是要做的事情并不相同。一学期有两次论文，那段时间我就把`- 整理案例库 @杨老师`这样的任务添加进 TaskPaper；而演示无聊的 PPT 时，我的 Daily Plan 里就会多出一堆东西：

```
- 讲稿 @Minja
- 设备调试 @Minja
- 主讲 @Minja
- 答辩 @Minja
- 板书 @Minja
```

为什么成员都是我自己？我觉得其他人效率太低，所以在课堂上从不和那些对于一个 PPT 是做成 4:3 还是 16:9 好的「电脑专家」合作，当然我是一个非常喜欢 co-work 的人，只是分情况。TaskPaper 的意义在于细化了每一堂课，把抽象的课程表（它们不过是一堆色块）拆成了具体任务。我不喜欢班上那些家伙的一个原因就是他们根本不做任务管理，脑子里只有「我该上课了」，压根不知道自己上课应该干什么。我尝试合作过，荣幸获赞一句「能者多劳」，最后变成那个唯一在做事情的可怜虫。

在第一章里，我批评过日历不适合做任务管理，其实它应该拿来做日程的可视化呈现，尤其是那些铁打的、不容易改动的任务。**但是我更希望得到一份具体、可执行的 List，所以每天都把这些行程拆成任务，融合进 Daily Plan 里。**

### 过滤出当日可行之事

虽然 TaskPaper 贴心地为用户准备了`@today`标签（甚至预设了「cmd+y」的快捷键），但是我不喜欢用它。实际使用中，很多人的任务管理是需要分项目进行的，比如你可能建立了「整理公司仓库」和「写季度财报」两个项目，每个里面各有几条任务；由于工作量繁重，需要分成几天甚至半个月来完成，你用`@start`和`@due`标签给每个任务标好了起止时间。**但是你想过滤出当天的任务时，却发现只能使用`@due[d]=today`语法，根本没有`@today`的事……**

【@today 标签】

因此对我而言这个标签的意义不大。何况我相当一部分任务在八个小时前连影儿都没有——它们来自睡前阅读，没有进行过时间规划，不适合通过预定时间来过滤；所以我通过另一种过滤机制来汇集当日任务：**不分项目地把任务记在一个列表里，通过当日的实际情况选出可行的任务。**例如，`- Return：数据线 @Teresa`，就需要问清当天 Teresa 在不在学校，如果她去外出实习了，这个任务今天就不能执行；此外，我因为兼职接一些外景摄影的活儿，每天都要用屡试不爽的`@if(sun)`标签则标示出那些「靠天吃饭」的任务。

当然了，把任务记在一个列表里，这只限于任务不那么复杂、上升不到项目程度的情形。像花费时间不过几十分钟的宿舍换季大扫除，我就没必要把它列入单独的项目。对于稍稍复杂的任务，譬如写两三千字的稿子，大致分为写稿、配图和画题图三步，它们可以用子任务的形式列在写稿任务下面，也无需特意放在 sidebar。**项目和任务只是名字，不用拘泥，具体用哪个格式，一定程度上看你的实际情况。**

说白了，标签最后都是为过滤服务的。它的作用是标示任务进行时的情景，以助于我们过滤、筛选，从大饼一摊似的任务列表里选出真正能执行的。而对标签的认识，至此就算入门了。

### 输出

【Daily Plan】

通过以上的一番收集、筛选，我得到了一份质量更高的列表。它不再是流水账一样记下所有「想做的事」的幻想清单，而是可行的一份当日规划。我一般把这份 TaskPaper 保持打开，一直放屏幕左上角，以便时不时去检查一下进度。

除了给自己看的任务，我也做了一个特殊的共享列表。其实上文提到的`- Return：数据线 @Teresa @due(today 11:00)`（完整版）任务我就是和 Teresa 小姐共享的，算是提醒她一下，当天中午我要归还数据线，如果可以的话请尽量待在学校。等等，把这么一串很 Geek 的字符发过去，怕不是要被当成自以为是的傻瓜？其实 TaskPaper 可以直接导出任务到系统提醒事项，我得以把一些涉及其他成员的任务共享出去（还是卖个关子，在最后几章讲）。虽然我不需要，但不代表别人不用提醒，TaskPaper 考虑了普通人的习惯，给了一个方便的选择。

【导出提醒事项】

给自己看的、给别人看的列表都有了，就可以开始执行了。但我也必须正视现实，那就是列表里的任务不一定可以全部勾掉（事实却是我基本可以超额完成……我喜欢挑战，但不喜欢定毫无必要的目标）。极少数完不成的任务就顺延到后一天，再做一次 Daily Plan，反复问自己：

**我为什么没有完成它？我能完成它吗？我真的要做这件事吗？**

然后做出存留的选择，把时间花在刀刃上。所以 Daily Plan 不是虚假的承诺，而是深刻的反思和行动的指明灯；在我斗志昂扬地从床上跳下的时候，我就先把灯塔造好。
