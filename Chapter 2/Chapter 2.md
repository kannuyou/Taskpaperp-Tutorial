# 第二章：简单使用

> A simple to-do list app for Mac

名符其实，TaskPaper 兼具文本列表的易用和任务管理的强大。

它本质上就是一个对列表做了优化的文本编辑器，绝大多数操作用快捷键和打字进行更为高效便捷。从本章的基础操作开始，我主要基于这两点展开讲解。

【界面】

TaskPaper 的界面简洁，大致分为菜单栏（menubar）、侧边栏（sidebar）、搜索栏（searchbar）和编辑界面。考虑到 Taskpaper 界面为英文，后文相关位置都将用英文表示。

Menubar 内的选项非常丰富，不过出于效率考虑，我建议多用快捷键，大多数操作是有快捷键的（只要是能用键盘完成的操作，我都会在本教程里注明）。

左侧为 sidebar，所有的项目、标签和已保存的搜索项都在罗列于此，一览无余。

编辑界面上方是 searchbar，用于过滤。如果你没看见它，可以用`⌘command+⇧shift+F` 召唤出来。**在 High Sierra 中，召唤 searchbar 后光标不会自动定位，需要连按两次组合键，或者手动点选。**

## 基本元素
 
TaskPaper 的基本元素有四个，任务、项目、标签、备注。各呈不同的视觉效果，很好区分：

【【【基本元素截图】

- 任务：一根短线后面跟着一串文字，就是最普通的任务；
- 项目：颜色稍深的，就是项目；
- 标签：颜色最淡的则是标签；
- 备注：最后一个元素——备注——颜色与任务一样，但字体是斜体。

上手的时候，只需要使用它们就足够了，但我们仍有必要知道各个元素的标准用法。

### 任务

任务是 TaskPaper 里最基本的单位。

在一行文字前标出一个 `短横＋空格` 即设此行为任务，和我们在纸笔上的习惯很接近。

【【【操作】

快捷键为 `⌘command+return`，如果光标在任务行的末尾，**直接回车**也可以新建任务。

任务之下可设子任务，以缩进的的样式，形成任务组。

【【【操作】

并列的两个任务，使用快捷键 `⌃control+⌘command+→` 将下方的任务右移一级，便把它设为了上方任务的子任务。移动光标到主任务左侧，会出现空心小红点，点击可折叠/展开子任务列表，它的快捷键为 `⌘command+9/0`。已折叠的任务左侧红点会变成实心，以示其下包含有的已折叠项。

【折叠】

### 项目

在一行文字后标出一个 `冒号` 即设此行为项目。项目下可以添加任务，操作与设置子任务类似。所有的项目都会显示在 sidebar 的「Project」栏内，包括子项目（它们可以折叠起来）。

【【【操作】

TaskPaper 里有一个特殊项目：Archive。使用快捷键 `⌘command+⇧shift+A` 可以将带有 `@done` 标签（关于标签详见下一节）的任务归档到 Archive 项目，从此这些任务不再显示于任务列表，只能在 Archive 里看到它们。如果你不想看到已完成的任务，就可以用这个方式快速归档它们。

【Achieve】

在几个项目之间移动项目（或任务），只需要光标拖拽：按住它们前面的红点，把它拖到你想要的地方——编辑界面、sidebar，或者另一个 TaskPaper 文档都可以。

【拖拽项目】

当然，因为 Taskpaper 的纯文本机制，剪切粘贴也可以。

### 备注和附件

如果输入一行文字，不做任何处理，即设此行为备注。对于任务、项目的大段描述，可以用备注的形式添加进 TaskPaper。

至于附件，它本不属于 TaskPaper 的基本元素，但是也起到了附加信息的作用，我把它和备注一起介绍。TaskPaper 里的附件不是真把原文件塞进来了，它只是显示一个可点击跳转的完整路径。

添加附件很方便，直接把目标文件拖拽到 TaskPaper 页面即可。

【拖动添加文件】

如果你的备注过长，或者附带了一串附件路径，建议把它移到对应任务的下一层级（快捷键 `control+⌘command+→`），

【【【移动层级】

如此一来，平时可以把它折叠起来。折叠的快捷键为 `⌘command+9`，展开为 `⌘command+0`。

【折叠备注】

### 标签

`@` 符号后面的文字即为标签内容，在默认主题里显示为灰色字体，以示区分。一个普通标签里不能出现空格和 emoji 表情。所有标签都会显示在 sidebar 的「Tags」内，点击可快速查看所有带该标签的任务/项目。

【【【操作】

TaskPaper 的文字标签支持灵活的赋值、和运算号组合，进行精确的过滤。过滤对于任务管理是颇具意义的，你可以无视层级，用一个或一组搜索条件从纷杂的列表中快速揪出需要的任务，聚焦于这些事情，不受其他任务打搅。如果任务管理工具只是摆路边摊一样，把任务、项目和标签一股脑堆在我眼前，我反而不知道自己该干什么。

TaskPaper 的标签是纯文字的，也许你会暗自抱怨：居然还要打字，实在是太不方便了。实际上，TaskPaper 的标签支持自动补全，不用次次完整打出。你输入过一次标签之后它会被记忆住，下次打出几个首字母就能显示出来。如果你连 `@` 也不想打，可以使用 `⌘command+T` 呼出标签栏，同样享受自动补全。

【自动补全】

一般来说，你可以往标签里塞任何文本内容，不会引起程序曲解；但是 TaskPaper 预设了一些标签，它们具有特殊的意义，需要单独拎出来介绍。

#### `@done`

把一个任务标记为已完成的状态，可以像其它任务管理软件一样，点击最左侧的短横，任务将被画上删除线、标为已完成状态。如果你想用快捷键直接完成一个任务，可以用 `⌘command+D`。左侧 serachbar 内置了一个搜索项 `not @done`，可以快速过滤出没有完成的任务。如果误按，再按一次快捷键可以将任务重置为未完成的状态。

在 TaskPaper 的 Task 设置里，可以勾选「include date when tagging task with @done」，这么做可以为已完成的任务附上日期，方便日后做详细的回顾。

【两种 done】

#### `@due`和`@start`

它们分别标注任务的截止日期和开始日期。单独的 `@due` 或者 `@start` 没有什么意义，它们需要配合一个时间点才能起到指示任务具体执行时间的目的。**TaskPaper 用 ISO 8601 格式的文本存储时间，你不用搞懂这具体时什么，只需要知道这是一个超棒的通用时间标准，在最后几章的实例中你会看见 TaskPaper 如何导出时间到别的应用里。**但是用该格式输入完整的文本并不方便，为此，你可以使用 TaskPaper 预设的日期&时间语法。日期、时间和 `@due` 或 `@start` 标签组合起来就是这样：

```
- 订阅 Ulysess @due（2017-10-22）
- 跑步五公里 @start(2017-10-02)
- 阅读《睡鼠说》@due(today) @start(next Sunday)
```

这两个标签也在 sidebar 上，不过作用有限，只能过滤出「有截止日期的任务」或「有开始日期的任务」。需要精确地过滤一组符合要求的任务出来，还需使用下一章详细介绍的搜索语法。

别紧张，会有模板给你。

#### 日期语法

鉴于接下来许多标签都用到了 TaskPaper 格式的日期与时间，有必要讲一下它们的语法。TaskPaper 里的日期和时间有两种类型：绝对日期和时间，形如「2017-9-15」「9:00 am」；相对日期和时间，比如「next Wednesday」「+2h」。你也可以组合使用它们，像是「today+24h」。

【日期时间语法】

TaskPaper 的日期和月份可以使用缩写，例如 Saturday 缩写为 Sat。一天的开始是午夜，一周的起始是周日（依据 ISO 8601 标准）。

需要注意的是，相对时间的「坐标点」是当下或当天，而非录入任务时，也就是说，你五个小时前标上的 `+5h`，现在它指向的绝对时间还是五小时后，看图比较容易理解：

所以我建议打标签的时候少单独使用相对时间，多使用绝对时间或把它们组合起来使用，避免歧义。

## 多文档管理

TaskPaper 可以打开多个窗口或者标签页。

对于同时管理多个复杂项目的使用者，我建议他们把几个项目存储在不同的 TaskPaper 文档里，所需搜索和定位时候的范围。

## 实例：Daily Plan

> If a task isn’t written down in a list or set to alert me in iCal, it’s gone. Forgotten. Doesn’t get done.（你不把任务记下来或设个闹钟，它就溜走了、被遗忘了，不会被完成）——Marco Arment（Instapaper、Tumblr 开发者）

任务管理最基础的就是「记录」，无论是用原始的纸质便笺，还是灵活的任务管理应用。了解完 TaskPaper 的基本元素，我通过一个例子来展示它们的简单运用，也阐述我做任务管理的基本思路。

一开始的时候，我对「管理」也并不精通，于是就从「记录」入手；恰好这在许多认为管理体系里都是一个容易做而且用处大的步骤，GTD 中就把「收集」作为第一步，让践行者清空大脑、把任务一个不漏地记在纸上（当然现在我们是记在 TaskPaper 里）。把任务从大脑里腾到纸或其他载体上，有两个优点：

* 不容易遗忘
* 集中注意力

而这一切的意义在早晨更为重大。许多人认为「一日之计始于晨」，对此我有着独特的理解：**在任务管理中，早晨的地位不是开始，而是过渡。**任务管理的周期并不和太阳的起落完全吻合，日出而作、日落而息在今天看来有些落伍（对于养生达人来说也许不是这样），**在我看来早晨的作用是回顾昨日任务、规划当日任务，做一个 Daily Plan，而 TaskPaper 就是快速、有序、结构化完成这一切的利器。**

记录的时候，需要一边打字一边筛选、评估，让最后出来的 Daily Plan 具有较高的可行性。

Daily Plan 里的任务，不是起床后从天而降、凭空捏造的，它们有三个来源：

* 昨日余留任务；
* 躺在床上冒出的灵感；
* 循环任务；

所以在这个来自现实的例子里，我一开始就会给你看一堆任务，而不是假惺惺手把手地教你新建。

### 处理遗留

【历史遗留】

我知道，许多人习惯熬夜，不做好明天的任务规划不睡觉。我也曾是一个夜猫子，但周围多数人的作息还是比较健康，慢慢我也习惯了早睡早起，一段时间后，我意识到早起后脑子更清醒，可以把遗留任务和其他来源的任务统筹考虑，不像临睡前的保证那样，缺乏实际意义地许诺「我明天一定会做」。我将展示如何用 TaskPaper 更好的完成它们。

讲具体操作之前，须知任务总是做不完的，相信效率越高、才华越是横溢的你越有这种感受：总能想到更多的事情、更棒的主意。这是好事，说明我们在任务管理中处于主动地位，没有被任务牵着鼻子走。下面这位兄台的心态值得学习，接受做不完任务的现实，把历史遗留记录下来，还有明天：

> I jot down all the ideas still in my head — all the loose ends I didn't tie up, all the things I may want to do but didn't get to. It can be hard to call it quits for the day when there are still things that can be done. And so this entry in my Day One is my way of admitting that the day is done and, yes, there is still work to do, but there is always tomorrow.——Shawn Blanc（知名科技博主）

话说回来，明天可不是拖延的借口。到了「明天」早上，就要好好地规划任务了。**我借助 TaskPaper 来对遗留任务做出必要性和可行性的评估：**需不需要做这件事？能不能够做这件事？

首先，直接扫一眼昨日任务列表，所有的遗留都在里面。取几个例子：昨日下课晚，水果店关门，我没有完成买苹果的任务，它理所当然顺延到今天；而取景的任务，已经足足拖延了半个月，这期间当地不是暴雨就是雾霾（接下来还是阴雨连连），完全没法拍摄。

着重看带有 `@due` 标签的任务，它们带有截止日期。取景已经拖了很久了，结合连日阴雨的现状，这个任务的可行性就存疑。经历了两年的每日清理，我发现一个残酷的事实：如果某个任务连续拖了一周，不是这件事毫无处理的必要，就是我根本没有能力去完成它（前提是执行者不是一个懒汉），无论何者，我都应该无视这个没有意义的任务，把精力放在更可行的事情上。

至于那些没有具体标签、备注的任务，像买一些食品、生活用品等琐事，这些任务没完成……很可能是你真的有拖延症前兆哦。

### 收集灵感和细碎片段

灵感总是不期而至，甚至躺在床上的时候也会袭来。我们需要抓住这些碎片，待翌日把它们变成可行的任务。

> 格雷厄姆是那种满脑子想的都是学问的学者，睡觉时还会在枕边放着一个记事簿。——「股神」巴菲特回忆恩师格雷厄姆

我也在床头放了铅笔和便笺，不过还有别的目的，即减少对睡眠的干扰，不愿点亮手机屏幕记录任务。灵感一来，就摸黑草草记下关键词，醒来后，这些堪比魏晋书法作品的纸片，就成了我的一大任务来源。接下来不是机械地把任务输入电脑，而是开始解构任务。

**我的任务基本是「动词+名词」的格式，像`- Search：《偷窥狂》`，这样的显示非常简明，并且记录也方便。**动词很神奇，它本身是带有**先后逻辑**的：我记录在纸上的关键词往往只有名词，而且是专有名词，并不记录动词；因为我到底是搜索、了解、下载还是观看这部电影，写便签时尚不确定，需要录入 TaskPaper 的时候统筹当天任务、再做判断。

我在长期的阅读中总结了一个很黑暗的规律：一本书里所提到的另一部作品（只要它不在「中学生必读书目」「人生必看 100 部电影」里），后者的名字在绝大多数情况下是译者兴之所至随意取的，我几乎没有直接成功搜到过。也由此，找资源将是一个不确定耗时、不一定能完成的任务，我添加任务的时候只敢用「Search」，而不是「Download」之类。至于修饰的词语我偶尔会用，比如说《论摄影》这本书，有桑塔格和亚当斯两位大师的同名作，我只在这种特殊情况下使用`- Search 《论摄影》苏珊`的格式。

但不是所有的任务都需要如此循规蹈矩。回忆一下我在第一章里的例子，用 TaskPaper 记录当日所需商谈的客户名单。由于我是个不关心公司其他事务的创始人，所有涉及公司的事情，基本就是被动地谈客户，那么「谈」这个动作就可以缺省了；原先属于备注的产品名，因为简短而具体，此时就代替客户名成为任务中的谓语对象。

**总之，我记录任务，一般只需要最核心的动作和对象。**如此，在 check 任务列表的时候，我的注意力可以集中在最重要的元素上，「Just do it」。用 TaskPaper 记录这样「格式化」的任务很棒——还是回到纯文本，它显示的方式直观而无多余元素，看着清清爽爽的列表我就感到动力十足。

> Keep a to-do list……A real one. One that you actually use and update throughout the day. It doesn’t need to be fancy, like the getting-things-done task managers — I use TaskPaper, which is essentially a text editor with optimized syntax highlighting for to-do lists, against a text file on Dropbox.（做个你真正在用的、每天更新的清单，它不用像 GTD 管理工具那么酷炫——我用的就是对于清单显示方式有优化的 TaskPaper，并把它存在 Dropbox 里）——Marco Arment

### 拆开「日复一日」的任务

【换例子】

【我把大量时间花在处理由灵感带来的任务上，但我相信这不是长久之计。我算半个搞艺术的（设计师总误认自己是艺术家，而后者很少以设计师自居），现在年轻，尚可一年到头智力火花四溅，可惜我不是天才，必须持续地学习和练习。

我在学校蹭了好几门需要的课程，上课就是每日「循规蹈矩」的任务。它们可以通过时间块的方式展现，比如写进日历、设置循环——不过我更加喜欢用 TaskPaper「分拆」它们。

例如每隔两天就有刑法课（来自我的主修专业），但是要做的事情并不相同。一学期有两次论文，那段时间我就把 `- 整理案例库 @杨老师` 这样的任务添加进 TaskPaper；而演示无聊的 PPT 时，我的 Daily Plan 里就会多出一堆东西：

```
- 讲稿 @Minja
- 设备调试 @Minja
- 主讲 @Minja
- 答辩 @Minja
- 板书 @Minja
```

为什么成员都是我自己？我觉得其他人效率太低，所以在课堂上从不和那些对于一个 PPT 是做成 4:3 还是 16:9 好的「电脑专家」合作，当然我是一个非常喜欢 co-work 的人，只是分情况。TaskPaper 的意义在于细化了每一堂课，把抽象的课程表（它们不过是一堆色块）拆成了具体任务。我不喜欢班上那些家伙的一个原因就是他们根本不做任务管理，脑子里只有「我该上课了」，压根不知道自己上课应该干什么。我尝试合作过，荣幸获赞一句「能者多劳」，最后变成那个唯一在做事情的可怜虫。】

在第一章里，我批评过日历不适合做任务管理，其实它应该拿来做日程的可视化呈现，尤其是那些铁打的、不容易改动的任务。**但是我更希望得到一份具体、可执行的 List，所以每天都把这些行程拆成任务，融合进 Daily Plan 里。**

### 着重处理当日可行之事

在所有的列表里面，最重要的就是「今天」，我之前所做的三步，其实最后就是汇集成一份今日任务列表。《GTD》里面把它视为处理任务的站点，Todoist、Things 等优秀的任务管理工具专门的「今天」，TaskPaper 则通过标签来过滤出今日任务。它自带了两种表示「今天」的标签，我们一一来看。

第一种是简单的 `@today` 标签，sidebar 里默认就有它。TaskPaper 还为它设置了快捷键 `⌘Command+Y`，这就方便了我快速选出今日任务：检视一遍整个收件夹，看到适合今天完成的，就打上今日标签，很顺手。`@today` 就像一个小星标，提醒你这个任务今天得完成。

【【【⌘command+Y】

第二种是使用 `@start` 或 `@due` ，或者把它们结合起来，表示任务的时间跨度。比如 `- 写检察院实习报告 @start(2017-09-25) @due(2017-10-08)` 这样一条任务，它的跨度就是九月底到十月初，如果今天恰好在这个区间内，那么这个任务就属于「今天」了。你一定想问，那怎么过滤出这个任务呢？本章暂不介绍复杂的过滤语法，我们直接使用 sidebar 里的提供的「Today」，点一下即过滤完毕。它还把带有 `@today` 标签的任务也归进来了，你不会漏掉方才手动打标签的任务。

【【【Today】

此外，我还有第三种具有个人特色的今日任务：**符合当天条件的任务**。例如，`- Return：数据线 @Teresa`，就需要问清当天 Teresa 在不在学校，如果她去外出实习了，这个任务今天就不能执行；此外，我喜欢趁天气好的时候去后山的溪涧中采风，常常用屡试不爽的 `@if(sun)` 标签则标示出那些看老天脸色的任务。使用条件标签的好处是可以把任务记在各个列表里，再通过当日的实际情况选出可行的任务，不需要一开始就让它们攻占宝贵的今日列表。

【导出提醒事项】

给自己看的、给别人看的列表都有了，就可以开始执行了。但我也必须正视现实，那就是列表里的任务不一定可以全部勾掉（事实却是我基本可以超额完成……我喜欢挑战，但不喜欢定毫无必要的目标）。极少数完不成的任务就顺延到后一天，再做一次 Daily Plan，反复问自己：

**我为什么没有完成它？我能完成它吗？我真的要做这件事吗？**

然后做出存留的选择，把时间花在刀刃上。所以 Daily Plan 不是虚假的承诺，而是深刻的反思和行动的指明灯；在我斗志昂扬地从床上跳下的时候，我就先把灯塔造好。
